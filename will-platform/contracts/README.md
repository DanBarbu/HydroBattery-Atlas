# WILL Plugin Contracts

> Crown jewels. ADR-002. Treat as a public product.

## Versioning

- Major versions are immutable. Within a major version, additive changes only; new fields must be optional.
- The platform supports the current major **and** the previous major for a minimum of three years.
- A plugin declares its contract version in `DescribeResponse.contract_version` (e.g. `v1.0`).
- The plugin loader rejects plugins whose contract version is unknown.

## Generating stubs

Stubs are generated by CI (`buf generate`) and published to the internal artefact registry. Local generation:

```bash
cd will-platform/contracts
buf generate
```

## Conformance tests

Every contract change runs the previous-major reference plugin against the new core in CI. A breaking change must bump the major version and is reviewed by the Tech Lead + Security Engineer + Compliance Officer.
